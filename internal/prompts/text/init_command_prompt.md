生成一份名为 AGENTS.md 的文件，作为本仓库的贡献者指南。
目标是写出清晰、简洁、结构化的文档，每个部分都要有描述性标题和可执行的说明。
可参考下方大纲按需增删——相关内容可补充，不适用的可省略。

语言与输出

- 输出语言优先与用户请求或仓库主要文档语言一致；无法判断时默认使用简体中文。
- 需要多语言时，以中文为主，并在同一标题或要点后补充其他语言对照，确保中文单独阅读也可用。
- 章节标题与正文都遵循上述语言偏好。

文档要求

- 标题使用 “Repository Guidelines”（中文场景可附加“仓库指南”等译名）。
- 使用 Markdown 标题（#、## 等）组织内容。
- 保持简洁，200-400 字为佳。
- 说明保持简短直接，聚焦本仓库。
- 需要时给出示例（命令、目录路径、命名模式）。
- 语气保持专业、指导性。

推荐章节

项目结构与模块组织

- 概述项目结构，标明源码、测试与资源的位置。

构建、测试与开发命令

- 列出构建、测试、本地运行的关键命令（如 npm test、make build）。
- 简要说明每条命令的作用。

编码风格与命名约定

- 说明缩进规则、语言特定的风格偏好和命名模式。
- 包含使用的格式化或 lint 工具。

测试规范

- 说明测试框架与覆盖率要求。
- 说明测试命名惯例以及如何运行测试。

提交与 PR 规范

- 总结 Git 历史中的提交消息惯例。
- 说明 PR 需要的内容（描述、关联 issue、截图等）。

（可选）如有需要，可补充安全与配置提示、架构概览或 Agent 专属说明。
